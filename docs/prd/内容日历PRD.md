## 1. 背景与目标

### 1.1 模块定位

**内容日历 = 任务执行统一入口**

```
收到任务通知 → 内容日历看到任务 → 点击执行 → 完成
```

**核心职责：**

| 做                           |
| --------------------------- |
| 展示被分配的任务                    |
| 追踪执行进度                      |
| 任务执行操作                      |
| 创建任务（探盘/话术训练/录音分析/文案生成/自定义） |

**入口说明：**

- 内容日历：执行任务 + 创建任务（统一入口）
- 个人中心：查看我创建的所有任务（单独管理页）

**两者关系：**

| 入口   | 位置    | 功能                  |
| ---- | ----- | ------------------- |
| 内容日历 | 首页Tab | 执行被分配的任务 + 创建各种类型任务 |
| 个人中心 | 底部Tab | 查看我创建的所有任务          |

### 1.2 需求背景

**为什么要做内容日历？**

- 经纪人需要统一的入口查看和执行被分配的任务
- 新盘上线、话术训练等通知需要集中展示
- 任务执行进度需要实时追踪

**当前问题**

- 任务通知分散，用户容易遗漏
- 缺少统一的移动端任务执行入口
- 无法实时追踪执行进度

### 1.3 目标

| 目标   | 说明                           |
| ---- | ---------------------------- |
| 统一入口 | 所有任务类型（探盘、话术、录音、文案）统一在内容日历展示 |
| 消息聚合 | 新盘通知、系统通知、团队通知统一入口           |
| 移动执行 | 小程序端完成探盘/话术/录音/文案任务执行        |
| 进度追踪 | 实时查看任务执行进度（未开始/进行中/已完成/超时）   |

```
作为 经纪人
我希望 收到任务通知后能快速进入执行

Given 经纪人收到任务推送通知
```

## 3. 功能概述

### 3.1 核心功能

| # | 核心功能     | 说明               |
| - | -------- | ---------------- |
| 1 | **任务展示** | 内容日历统一展示所有被分配的任务 |
| 2 | **消息通知** | 新盘通知、系统通知、团队通知聚合 |
| 3 | **任务执行** | 探盘/话术/录音/文案任务执行  |
| 4 | **进度追踪** | 查看执行进度和催办        |

### 3.2 任务类型

| 任务类型   | 执行方式  | 阶段说明          |
| ------ | ----- | ------------- |
| 探盘任务   | 移动端执行 | 3阶段（路书→文案→视频） |
| 话术训练任务 | 移动端执行 | 单一阶段          |
| 录音分析任务 | 移动端执行 | 单一阶段          |
| 文案生成任务 | 移动端执行 | 单一阶段          |

### 3.3 内容日历入口结构

```
小程序底部Tab
├── 首页（内容日历）
│   ├── 日历视图（按日期展示任务）
│   ├── 任务卡片列表
│   ├── 统计卡片（待办/进行中/已完成/超时）
│   ├── 筛选功能（类型/状态）
│   └── ➕ 创建任务入口 ← 在这里创建各种类型任务
│
├── Claw（AI助手）
│
├── 服务工具
│
└── 个人中心
    └── 我的任务（查看我创建的所有任务）
```

### 3.4 探盘任务结构

探盘任务包含**多个子任务**：

```
探盘任务
│
├── 探盘路书（必选）
│   ├── 阶段1：信息收集
│   ├── 阶段2：文案生成
│   └── 阶段3：视频生成（后台异步）
│
├── 话术训练（可选）
└── 录音分析（可选）
```

**完成规则**：

| 规则     | 说明                          |
| ------ | --------------------------- |
| 探盘路书完成 | 3个阶段（信息收集→文案生成→视频生成）文案确认即完成 |
| 话术训练   | 如果开启，则必须完成，按照配置的训练次数、评分要求   |
| 录音分析   | 如果开启，则必须完成，按照配置的上传数量        |
| 任务完成   | 所有子任务都完成                    |

**子任务关系**：

| 关系   | 说明           |
| ---- | ------------ |
| 探盘路书 | 必选，3个阶段顺序执行  |
| 话术训练 | 可选，完成后计入任务完成 |
| 录音分析 | 可选，完成后计入任务完成 |

### 3.5 任务卡片显示内容

每张任务卡片展示内容：

| 字段    | 说明                    |
| ----- | --------------------- |
| 任务名称  | 如"中海汇德里探盘"            |
| 任务类型  | 如"探盘"                 |
| 发起人   | 谁创建的任务                |
| 截止时间  | 任务截止日期                |
| 剩余时间  | 剩余X小时 / 已超时X天         |
| 当前状态  | 待开始 / 进行中 / 已超时 / 已停止 |
| 总进度   | X/X 完成（文字+进度条）        |
| 子任务列表 | 各自状态（探盘路书/话术训练/录音分析）  |

**任务卡片布局**：

```
┌─────────────────────────────────────┐
│  [探盘] 中海汇德里探盘         ●进行中 │
│  发起人：张三                      │
│  截止：04-20 18:00  剩余2小时      │
│  ─────────────────────────────────── │
│  总进度：2/3 完成                   │
│  ▓▓▓▓▓░░░░░░░░░░░░░░░            │
│  ─────────────────────────────────── │
│  ✅ 探盘路书 → 文案生成 → 视频生成  │
│  ⏳ 话术训练                       │
│  🔒 录音分析（未开启）              │
└─────────────────────────────────────┘
```

**卡片点击行为**：

| 状态          | 点击后                |
| ----------- | ------------------ |
| 待开始         | 进入执行页面 + 状态变为"进行中" |
| 进行中         | 进入执行页面（继续操作）       |
| 已完成/已超时/已停止 | 进入详情页面（只读）         |

**重要规则**：

| 规则        | 说明                     |
| --------- | ---------------------- |
| 待开始 → 进行中 | 点击任务卡片（没有子任务）/子任务时自动变更 |
| 自动保存      | 执行过程支持中断后继续            |
| 文案确认完成    | 完成文案生成后 → 任务完成         |
| 任务停止      | 任务被停止 → 所有执行人任务变为"已停止" |

## 4. 任务执行详细设计

### 4.1 探盘任务执行

#### 4.1.1 任务状态

| 状态  | 说明                       |
| --- | ------------------------ |
| 待开始 | 任务已分配，等待执行人开始            |
| 进行中 | 执行人正在执行（至少完成了1个阶段）       |
| 已完成 | 执行人已确认文案，任务完成（视频在后台继续生成） |
| 已超时 | 超过截止时间还未完成               |
| 已停止 | 任务被创建者/管理员停止，所有执行人任务停止  |

#### 4.1.2 阶段状态

| 阶段状态 | 说明         |
| ---- | ---------- |
| 未开始  | 该阶段还未开始    |
| 进行中  | 该阶段正在执行    |
| 已完成  | 该阶段已完成，已锁定 |

#### 4.1.3 任务详情页

```
┌─────────────────────────────────────────┐
│  < 返回     探盘任务详情           ••• │
├─────────────────────────────────────────┤
│  🏠 中海汇德里                         │
│  📍 浦东新区张江高科技园区             │
│  ⏰ 2026-04-08 18:00 (已超时)        │
├─────────────────────────────────────────┤
│  完成进度                        1/3   │
│  ████████░░░░░░░░░░░░░░░░░░░░░░░░    │
│  ●────────●────────●                   │
├─────────────────────────────────────────┤
│                                         │
│  ┌──────────────────────────────────┐ │
│  │ ① 信息收集                       │ │
│  │ ✅ 已完成 | 10:30:00      [进入]│ │
│  └──────────────────────────────────┘ │
│  ┌──────────────────────────────────┐ │
│  │ ② 文案生成                       │ │
│  │ 进行中                    [进入]  │ │
│  └──────────────────────────────────┘ │
│  ┌──────────────────────────────────┐ │
│  │ ③ 视频生成                       │ │
│  │ 未开始                    [进入]  │ │
│  └──────────────────────────────────┘ │
│                                         │
└─────────────────────────────────────────┘
```

#### 4.1.4 阶段推进流程

```
完成阶段1（信息收集）
    │
    ▼
执行人点击"提交素材"
    │
    ▼
⚠️ 提示：确认提交全部素材？
         提交后将进入【文案生成】阶段
         当前阶段不可更改
    │
    ├── [取消] → 留在当前阶段
    │
    └── [确认] → 进入阶段2（文案生成）
              │
              ▼
         阶段1状态变更为"已完成"且锁定
         阶段2状态变为"进行中"
    │
    ▼
完成阶段2（文案生成）
    │
    ▼
执行人点击"确认文案"
    │
    ▼
⚠️ 提示：确认文案？
         确认后任务将标记为完成
         视频生成将在后台继续进行（约30分钟）
    │
    ├── [取消] → 留在当前阶段
    │
    └── [确认] → 任务完成
              │
              ▼
         阶段2状态变更为"已完成"
         阶段3（视频生成）在后台异步进行
```

### 4.2 探盘任务子任务执行

#### 4.2.1 话术训练

| 配置项    | 说明      |
| ------ | ------- |
| 开启话术训练 | 开关，默认关闭 |
| 训练项目   | 开启后需选择，从项目列表多选  |

**联动规则**：选择训练项目后，需配置完成标准

| 完成标准   | 说明        |
| ------ | --------- |
| 训练次数   | 完成X次训练才算完成   |
| 单次评分   | 单次评分达到X分才算完成 |

**执行流程**：选择训练项目 → 配置完成标准 → 开始训练 → 达到完成标准 → 完成

#### 4.2.2 录音分析

| 配置项    | 说明      |
| ------ | ------- |
| 开启录音分析 | 开关，默认关闭 |
| 上传录音数量 | 开启后需配置，最少1个  |

**执行流程**：上传录音 → AI分析 → 查看结果 → 完成

#### 4.2.3 文案生成

文案生成作为独立任务类型时：

**执行流程**：选择楼盘 → 配置生成参数 → AI生成 → 确认/修改 → 完成

## 7. 验收标准

### 7.1 功能验收标准

| 功能点    | 验收条件                                           | 优先级 |
| ------ | ---------------------------------------------- | --- |
| 任务展示   | Given 经纪人被分配任务 When 打开内容日历 Then 任务卡片展示正确       | P0  |

| 指标     | 目标值    |
| ------ | ------ |
| 页面加载时间 | <2秒    |
| 推送送达率  | ≥95%   |
| 列表查询响应 | <500ms |

### 8.1 端职责
