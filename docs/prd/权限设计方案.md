## 一、角色定义

### 1.1 管理侧

| 角色 | 说明 | 权限范围 |
|-----|------|---------|
| **超级管理员** | 最高权限 | 全局：所有数据、所有操作 |
| **管理员** | 运营/策划 | 全局：可创建系统任务/通知、可看所有数据 |

### 1.2 用户侧

| 角色 | 说明 | 权限范围 |
|-----|------|---------|
| **区域经理** | 团队负责人 | 辖区：可创建用户任务、可看团队数据 |
| **经纪人** | 执行者 | 个人：只能看自己数据、只能给自己创建任务（话术训练、录音分析） |

### 1.3 组织架构

```
超管/管理员
    │
    └── 按实际组织架构配置
            │
            ├── 区域A
            │     ├── 区域经理A
            │     │     ├── 经纪人A1
            │     │     └── 经纪人A2
            │     │
            └── 区域B
                  └── 区域经理B
                        ├── 经纪人B1
                        └── 经纪人B2
```

**规则**：不限制层级深度，按实际组织架构选择执行人

## 二、任务创建权限

### 2.1 权限矩阵

| 角色 | 可创建任务类型 | 执行人选择范围 | 创建入口 |
|-----|------------|--------------|---------|
| 超级管理员 | 所有类型 | 所有人 | 管理后台 |
| 管理员 | 系统任务/通知 | 所有人 | 管理后台 |
| 区域经理 | 用户任务（探盘/话术/录音/文案/自定义） | 自己 + 下级 | 小程序 |
| 经纪人 | 有限制（见下方说明） | 自己 + 下级 | 小程序 |

### 2.2 经纪人创建任务限制

| 可创建的任务 | 限制条件 |
|-------------|---------|
| 话术训练任务 | 直接创建 |
| 录音分析任务 | 直接创建 |
| 探盘任务 | ❌ **无权限** |

**无权限提示**：`"请联系团队负责人进行任务创建"`

### 2.3 执行人选择规则

| 创建者 | 可选择的执行人 | 范围限制 |
|-------|---------------|---------|
| 管理员 | 所有人 | 无 |
| 区域经理 | 自己 + 下级 | 仅辖区人员 |
| 经纪人 | 自己 + 下级 | 仅下级（若有） |

**选择方式**：按组织架构树形选择，支持搜索，不限制层级深度

## 三、系统任务与通知

### 3.1 系统任务

| 角色 | 可创建 | 说明 |
|-----|-------|-----|
| 超级管理员 | ✅ | 模板任务、全局下发 |
| 管理员 | ✅ | 模板任务、全局下发 |
| 区域经理 | ❌ | 无权限 |
| 经纪人 | ❌ | 无权限 |

**创建入口**：管理后台

### 3.2 消息通知

| 角色 | 可创建 | 接收配置 |
|-----|-------|---------|
| 超级管理员 | ✅ | 全局配置 |
| 管理员 | ✅ | 全局配置 |
| 区域经理 | ❌ | 接收自己辖区 |
| 经纪人 | ❌ | 接收个人 |

**配置项**：`接收消息通知`、`接收任务`权限按团队勾选

## 四、新盘通知

### 4.1 通知规则

| 层级 | 规则 |
|-----|------|
| **默认规则** | 所有团队负责人（区域经理及以上）收到通知 |
| **单独配置** | 单个楼盘可指定只通知某些团队 |

**优先级**：`楼盘单独配置 > 默认规则`

### 4.2 通知触发

```
新盘上线
    │
    ▼
查询楼盘的接收配置
    │
    ├── 有单独配置 → 只通知配置的团队
    │
    └── 无单独配置 → 通知所有团队负责人
```

### 4.3 小程序侧交互

```
收到新盘上线通知
    │
    ▼
展示在内容日历任务卡片中
    │
    ▼
点击查看详情
    │
    ├── 有创建探盘任务权限 → 显示"创建探盘任务"按钮
    └── 无权限 → 显示提示："请联系团队负责人进行任务创建"
```

## 五、任务追踪与催办

### 5.1 追踪视图功能

任务创建后可查看执行情况：

```
┌─────────────────────────────────┐
│         任务执行情况              │
├─────────────────────────────────┤
│  总执行人数：10人                │
│  ───────────────────────────────│
│  ✅ 已完成：3人                  │
│  ⏳ 进行中：4人                  │
│  📋 未开始：2人                  │
│  ⚠️ 已超时：1人                  │
├─────────────────────────────────┤
│  [查看详情]  [编辑任务]           │
│  [催办未开始]  [催办进行中]       │
└─────────────────────────────────┘
```

| 功能 | 说明 |
|-----|------|
| **查看详情** | 查看任务详细信息、执行进度 |
| **编辑任务** | 修改任务配置（阶段进行中不可编辑已完成部分） |
| **催办未开始** | 对未开始执行的执行人发送催办 |
| **催办进行中** | 对进行中但未完成的执行人发送催办 |

### 5.2 催办形式

| 通知方式 | 说明 |
|---------|-----|
| **小程序推送** | 执行人收到小程序内推送通知 |
| **短信（备选）** | 如小程序推送未送达，发送短信提醒 |

### 5.3 状态定义

| 状态 | 说明 |
|-----|------|
| **未开始** | 执行人还未开始执行任务 |
| **进行中** | 执行人正在执行中 |
| **已完成** | 执行人已完成全部阶段 |
| **已超时** | 超过任务结束时间还未完成 |

## 六、探盘任务阶段权限

### 6.1 三阶段定义

| 阶段 | 内容 | 流转规则 |
|-----|------|---------|
| **阶段1：探盘路书** | 采集内容（拍照/视频/文字） | 完成后用户选择是否进入下一阶段 |
| **阶段2：生成文案** | AI生成推广文案 | 完成后用户选择是否进入下一阶段 |
| **阶段3：生成视频** | 生成视频内容 | 最终产出，完成后任务结束 |

### 6.2 阶段配置项

```
探盘任务配置
├── 打卡范围配置
│   ├── 范围开关（默认开启）
│   └── 范围距离（N米）
│
├── 探盘路书配置
│   ├── 采集内容（多个点位）
│   ├── 生成文本内容开关（是否生成文案）
│   └── 生成多模态内容开关（是否生成图片/视频/语音）
│
├── 话术训练开关
│   └── 开启时需选择训练项目
│
├── 邀约录音开关
│   └── 开启时需填写上传录音数量（至少1）
│
└── 时间配置
    ├── 开始时间 ~ 结束时间（默认30天）
    ├── 提醒时间
    └── 催办时间（轻度/中度/重度）
```

### 6.3 阶段不可逆规则

```
阶段1完成 → 用户主动选择"进入下一阶段"
    │
    ▼
⚠️ 弹窗提示："确认进入【生成文案】阶段？确认后当前阶段不可更改"
    │
    ▼
[取消]  [确认]
    │
    ▼
确认后阶段1锁定，无法回退
```

## 七、数据查看权限

### 7.1 权限矩阵

| 操作 | 超管 | 管理员 | 区域经理 | 经纪人 |
|-----|-----|--------|---------|--------|
| 查看所有任务 | ✅ | ✅ | ❌ | ❌ |
| 查看辖区任务 | ✅ | ✅ | ✅ | ❌ |
| 查看自己任务 | ✅ | ✅ | ✅ | ✅ |
| 查看团队统计数据 | ✅ | ✅ | ✅ | ❌ |
| 导出任务数据 | ✅ | ✅ | 仅辖区 | ❌ |
| 删除任务 | ✅ | ✅ | ❌ | ❌ |

### 7.2 优先级

> **超管 > 管理员 > 区域经理 > 经纪人**

## 八、这期不做（避免复杂）

- ❌ 不做：给个人单独配权限（按团队配置）
- ❌ 不做：复杂规则（如按楼盘控制谁能发任务）
- ❌ 不做：权限模板
- ❌ 不做：经纪人创建探盘任务

## 九、总结

| 问题 | 解决方案 |
|-----|---------|
| 谁可以发任务 | 按角色+团队配置：管理侧可发系统任务，用户侧按权限发用户任务 |
| 谁可以看数据 | 超管/管理员看全部，区域经理看辖区，经纪人看自己 |
| 新盘通知发给谁 | 默认通知所有团队负责人 + 单个楼盘单独配置 |
| 执行人选择范围 | 按组织架构选择：只能选自己+下级 |
| 任务阶段控制 | 阶段不可逆，需用户确认后进入下一阶段 |
